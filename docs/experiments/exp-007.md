# DataRobot TS 二値分類 32kV閾値

## 1. 目的(Why)

DataRobot TS 回帰（exp-005）で MASE > 1（ナイーブ予測より劣る）という問題を受け、
予測タスクを「daily_max >= 32kV か否か」の二値分類に切り替えることで、
閾値超え検知精度を直接評価する。陽性率は全運転日の 6.1%。

## 2. データ・条件(Setup)

- 学習用 CSV: `outputs/dataset_ep370g_ts_binary_32kv.csv`
  - SHA256: `sha256:03294e0b76f377977c081c7a81579a79d22682eae9ba2b46074bcf03b5133294`
- ターゲット列: `exceeds_threshold`（daily_max >= 32kV → 1, それ以外 → 0）
- 陽性率: 6.1%（全運転日 119,055行中 約7,300行）
- 関連クリーニング: cleaning_ep370g_v2
- DataRobot Project ID: `69f0a6c14a723953d7a77d1a`
- DataRobot Project URL: https://app.datarobot.com/projects/69f0a6c14a723953d7a77d1a/models

## 4. 考察(Discussion)

- BT0 の AUC=1.0 は**信頼できない**。評価期間（2026-01-02 直前の7日間）に存在する
  プラグが 6 本のみであり、サンプル数が極端に少ない。
- 唯一まともな評価は BT1（LogLoss=0.253）。AUC 換算で概ね 0.85〜0.90 程度と推定される。
- BT2〜BT3 の異常値（LogLoss=6〜12）は回帰実験（exp-005: RMSE=9.5）と同一期間で
  発生しており、**2025年12月中旬以降のデータ急減が根本原因**。
  この期間は週間行数が 1,127 → 60 → 42 と激減しており、正常な評価が不可能。
- 特徴量重要度は `daily_max` の移動平均・ラグが支配的で、
  `operating_hours_since_exchange` は 5 位以内に入り劣化信号として機能している。
- 二値分類への切り替え自体は正しい方向だが、**データの時間的カバレッジ問題を
  解決しない限り評価の信頼性が低いまま**。

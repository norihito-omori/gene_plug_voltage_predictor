# 32kV 越境予知 — DataRobot TS vs ルールベース比較評価

## 1. 目的 (Why)

プラグ交換の主判断基準である「`daily_max >= 32kV` の初回越境」を事前に検知できるか、2 つのアプローチを定量的に比較する。

- **DataRobot TS 回帰** (v5): `daily_max` を FD=1〜7 で予測し、`prediction >= 32kV` をアラートとする
- **ルールベース**: 30 分粒度から集計した `hours_at_31kV` の 7 日累計が閾値を超えたらアラートを出す

### `hours_at_31kV` の定義

30 分粒度データから以下を計算し、日次データセットに結合した特徴量。

```
hours_at_31kV = (稼働中かつ 要求電圧 == 31kV のスロット数) × 0.5 [時間/日]
```

アラート条件: `hours_at_31kV の N 日ローリング合計 > 閾値 T`

### 3-4. 誤報 (FP) の性質

FP（越境しなかったサイクル中にアラートが出た件数）は EP370G 18/54件 (33.3%)、EP400G 20/121件 (16.5%)。**「31kV 付近には届くが 32kV には越境しないサイクル」が相当数存在する**ことを意味する。実運用では「アラートが出たら要観察」程度の運用が適切。

### 3-5. 未警告イベントの原因分析

未警告イベント（EP370G 10件、EP400G 4件）は全て `max_7d_sum ≈ 0h`、すなわち **越境前の期間に `要求電圧=31kV` の稼働スロットが存在しない**。データに前兆シグナルが存在せず、どの手法でも検知は構造的に不可能。これがこのルールが達成できる recall の上限である。

## 4. 考察 (Discussion)

### DataRobot TS が機能しなかった根本原因

`daily_max` は 1kV 整数センサーの出力であり、31 → 32kV の遷移が「瞬間的な跳び」として記録される。回帰モデルが「32kV 未満を観測した状態で 32kV を予測する」ためには、実際には存在しない中間値（31.x kV）を推定する必要がある。モデルの最大予測値が越境直前でも平均 30.57kV にとどまり、全機台で `prediction >= 32kV` を達成できなかった。

### `hours_at_31kV` が有効な理由

30 分粒度のスロット数を集計することで、1 日あたり 0〜24 時間の**連続値**に変換される。`daily_max` の 1kV 壁を迂回できる唯一のソフトウェア的手段であり、実電圧が 31.x kV 付近に滞在する時間の増加が特徴量に表れる。

### DataRobot TS に `hours_at_31kV` を追加した場合の見通し

- FD=1〜7 という予測窓は変わらず、ルールベースの中央値先行日数（5〜6日）はすでに FD の上限付近
- 未警告の機台（前兆なし）は特徴量を追加しても変わらない
- 1kV 問題のため回帰の予測値が 32kV を越えない構造的な限界が残る

**DataRobot TS がルールベースを上回る可能性は構造的に低い。**

### 精度を根本的に向上させる唯一の手段

センサー分解能を **0.1kV 単位**に上げること。これにより:

1. `daily_max` が 31.x kV の段階で検出可能になる
2. モデルが 32.0kV 越境を確率的に予測できる
3. 現在「前兆なし」で取りこぼしている機台も検知できる可能性がある

## 5. 推奨アラートルール (結論)

**評価方針**: 見逃し（アラートなしで 32kV 越境 = プラグ焼損リスク）を最優先で回避。誤報（アラートありで越境せず = 要観察コストのみ）は許容。よって **recall 最大 → FP 最小** の順で最適化。

| 機種 | アラート条件 | Recall | FP率 | 先行中央値 | 見逃し原因 |
|-----|------------|--------|------|-----------|----------|
| **EP370G** | `hours_at_31kV の 3日累計 > 0.5h` | **91.2%** (103/113) | 33.3% (18/54) | 97日 | 10件は 31kV 滞在ゼロ（前兆なし） |
| **EP400G** | `hours_at_31kV の 21日累計 > 0.5h` | **85.7%** (24/28) | 16.5% (20/121) | 46日 | 4件は 31kV 滞在ゼロ（前兆なし） |

### 解釈

- **見逃しゼロは達成不可能**: 未警告イベントは全て「越境前に一度も 31kV で稼働しなかったサイクル」であり、センサー分解能が 1kV 整数である限りどのアプローチでも検知不能。
- **EP370G**: 3日という短いウィンドウで十分な理由は、31kV 稼働が始まった翌日〜数日で既にシグナルが立ち、先行日数は中央値 97日と長い。閾値 0.5h（30分スロット 1回）と低く設定することで感度を最大化。
- **EP400G**: 21日ウィンドウが有効な理由は、1日あたりの 31kV 滞在時間が短く累積しないと閾値を超えないパターンが多いため。FP率が EP370G より低い（16.5% vs 33.3%）のは、EP400G の非越境サイクルには 31kV 稼働自体が少ないから。
- **FP（誤報）の実態**: アラートが出ても 32kV に届かなかったサイクルであり、「31kV 付近に滞在したが劣化が進まなかった」ケース。要観察リストに載せる運用なら実害は軽微。

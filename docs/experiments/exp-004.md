# Phase 0 trend features 27機場

## 1. 目的(Why)

exp-002（quick モード・27機場, RMSE=0.763）をベースラインとして、
トレンド・統計特徴量 4列（`voltage_trend_7d` / `days_since_exchange` /
`daily_max_rolling_mean_7d` / `daily_max_rolling_std_7d`）を追加した場合の
RMSE 改善幅を計測する。

- 学習用 CSV: `outputs/dataset_ep370g.csv`
  - SHA256: `sha256:db980a3c1da9deacf759386f33ec20df05647ad72ecb2d7e71c8410de9458adf`

## 3. 結果(Results)

- 採用モデル ID: `69f05b6dbfb19db3cef59ec3`
- Blueprint: Light Gradient Boosting on ElasticNet Predictions 
- RMSE: 0.728968
- テスト予測 CSV: `outputs/gene_plug_voltage_ep370g_test_pred.csv`
  - SHA256: `sha256:95473049554de1d2db85463db1de1ae204ce3bbcda7dc8c40c5bc66d938f5c81`

## 4. 考察(Discussion)

- RMSE 0.729 は exp-002（0.763）より **0.034V 改善**（-4.5%）。トレンド・統計特徴量の追加が
  有効であることが確認された。
- ベストモデルは exp-002/003 と同じ LightGBM on ElasticNet。特徴量追加後も同ブループリントが
  上位を維持しており、アーキテクチャの再現性が高い。
- `voltage_trend_7d`（直近7運転日の傾き）と `daily_max_rolling_mean_7d`（移動平均）が
  短期トレンドを捉え、モデルの予測精度向上に寄与したと考えられる。
- `days_since_exchange`（世代経過日数）はプラグ劣化の時間軸を明示するため、
  gen_no だけでは表現できない連続的な劣化パターンを補完した可能性がある。
